VIRTUAL TELESCOPE PROJECT ‚Äî CONTEXT FOR AI
============================================

Interactive virtual telescope. A desktop viewer (index.html) renders the sky
with Stellarium Web Engine. A smartphone (telescope.html) acts as the physical
telescope ‚Äî you point it and see the result on the viewer. Communication is
via WebRTC through the Protobject framework.

Two extra pages can connect: Mapa.html (2D map, touch to send coordinates)
and Pelota.html (3D globe with physical sensors), both send coordinates to
telescope.html.


TECH STACK
----------
- Svelte 5 (runes: $state, $derived) + Vite 6 (multi-page build)
- Stellarium Web Engine (WASM, loaded as global script)
- Protobject (WebRTC framework, loaded as global script)
- Cesium.js (3D globe, CDN lazy-loaded)
- Flatpickr (date picker, CDN lazy-loaded)
- luxon (date/time), pako (decompression), Three.js/Globe.gl (Pelota only)
- Language: all UI text is in SPANISH

KEY DATA FLOWS
--------------

1. ORIENTATION (smartphone ‚Üí viewer)
   Gyroscope + RelativeOrientationSensor ‚Üí calibration (3s, 300 samples)
   Startup: full-screen overlay with countdown "Mant√©n el tel√©fono quieto"
     ‚Üí "Calibraci√≥n en 3... 2... 1..." ‚Üí "Calibrando..." ‚Üí "Calibraci√≥n completada"
   Three modes based on FOV:
     - relative (FOV > 0.8 rad): RelativeOrientationSensor quaternion ‚Üí Euler
     - gyro (FOV < 0.8 rad): Gyroscope integration with deadzone
     - dynamic (FOV < 0.02 rad): Precision gain + smoothing filter
   Sends updateView { h, v } throttled at 20ms
   Started in Telescope.svelte, active in BOTH simple and advanced modes.
   Debug overlay hidden by default. Long-press finder-frame 5s to show/hide.
   Click debug overlay to recalibrate.

2. ZOOM ‚Äî Simple Mode
   Slider (0-150) ‚Üí sliderToFov() exponential mapping ‚Üí sendThrottled updateFov

3. ZOOM ‚Äî Advanced Mode
   Lens buttons (fl: 40, 24, 10, 4) ‚Üí magnification = FOCAL_LENGTH/fl
   FOV = (100/magnification) * PI/180 ‚Üí sendThrottled updateFov
   Also computes blur target and turbulence max based on magnification.

4. FOCUS (Advanced Mode)
   Focus slider (0-10) ‚Üí blur = f(|currentBlur - blurTarget|, zoomLevel)
   Sends via seeingOption { target: 'focus', value }

5. LOCATION
   GlobePicker (Cesium) or Mapa.html or Pelota.html ‚Üí { lat, lon }
   ‚Üí getMagFromLonLat (light pollution) ‚Üí tzlookup (timezone)
   ‚Üí stores observerLat/observerLon ‚Üí applyLocation locally + send to viewer
   Auto pollution in menu: fetches pollution from current observerLat/observerLon

6. TIME
   DateTimePicker: Flatpickr ‚Üí ISO ‚Üí MJD ‚Üí updateDate to viewer
   onChange fires on day/time selection; onMonthChange/onYearChange also update sky
   Sync interval (500ms) pauses for 3s after any user interaction
   Speed controls: 0x, 1x, 10x, 60x, 3600x
   Viewer broadcasts syncTime every 300ms back to telescope.

7. SEEING OVERLAY (viewer side)
   WebGL shader simulating atmospheric turbulence.
   Controls: turbulence (0-10), turbulenceMax (1-10), speed (1-5),
   focus (0-100), saturation (50-150).
   Telescope sends seeingOption messages to adjust these.
   Turbulencia slider in menu (advanced mode only).


STORES.JS ‚Äî STATE REFERENCE
----------------------------
Constants:
  MIN_FOV = 0.000005, MAX_FOV = 3.228859, FOCAL_LENGTH = 1200
  DIAMETER = 100, TELESCOPE_TYPE = 2 (refractor)
  Throttle rates: FOV_SEND_MS=50, ZOOM_THROTTLE_MS=100, etc.
  STEL_BUTTONS = { constellations, atmosphere, landscape, azimuthal, equatorial, dss }

Mutable state (module vars with get/set exports):
  engine, bortle, eyepieceFl(25), sqmReading(20), citySqmReading(20),
  logFov, currentFov(3), currentBlur(5), currentTZ(-4), engineUTC, pollution(9),
  observerLat(-33.4489), observerLon(-70.6693)

Svelte writable stores:
  modes { simple: true, advanced: false }
  isLoading (boolean)
  isMenuOpen (boolean)


PROTOBJECT MESSAGE HANDLERS
-----------------------------

Viewer (index.html) handles:
  toggleEyepiece, updateFov, updateBlur, updateView, applyLocation,
  setSpeed, updateDate, setDatetimeInterval, clearDatetimeInterval,
  updatePollution, stellariumOption, noLenBlurry, yesLenNormal,
  seeingOption, simpleSettings, advancedSettings,
  requestSynchronizeData, requestSynchronizeSimpleZoom

Telescope (telescope.html) handles:
  syncTime, setSynchronizedData (registered by AdvancedMode),
  setSynchronizedSimpleZoom (registered by SimpleMode),
  sendCoordinates (from Mapa/Pelota, applies location + pollution + timezone)


STELLARIUM ENGINE
-----------------
Global: StelWebEngine (loaded from stellarium-web-engine.js + WASM)
Initialized via initializeStelEngine(isTelescope).
  isTelescope=false: loads extended catalogs (DSO, stars, solar system)
  isTelescope=true:  minimal catalogs for guidescope preview
Canvas: #stel-canvas (must exist in DOM before init)
Access: engine.core.observer (lat, lon, elevation, utc, yaw, pitch)
         engine.core.fov (field of view in radians)
At startup, viewer calls enableSimpleModeSettings() to show star/planet names.


ORIENTATION SENSOR CONFIG
--------------------------
Located in src/lib/orientation.js, exported as Orientation object.
  gyroFreq: 100 Hz, relFreq: 30 Hz, calibDuration: 3s
  fovThreshold: 0.8 (relative‚Üîgyro switch)
  dynamicThreshold: 0.02 (gyro‚Üîdynamic switch)
  dynamicGainMultiplier: 10.0
  dynamicSmoothingFactor: 0.15
  gyroDeadzone: 0.003

Uses RelativeOrientationSensor (not Absolute ‚Äî no magnetometer needed,
works indoors, no compass issues).

Calibration: startup shows full-screen countdown overlay in Spanish.
Subsequent recalibrations via debug overlay click (no countdown, just status).

Debug overlay: hidden by default. Toggle with 5-second long-press on
finder-frame in AdvancedMode. Shows FOV, pitch, yaw, mode, status.

MENU UX HIERARCHY
------------------
Menu is a full-screen overlay (z-index 50).

Level 0 ‚Äî Main menu:
  Header: [X close menu] [Ubicacion btn] [Fecha y Tiempo btn]
  Body (space-evenly distributed):
    - 3x2 grid of Stellarium options (toggle on/off, visual opacity feedback)
    - Contaminacion Luminica: slider (1-9 Bortle) + Auto toggle
      Auto: fetches pollution from observerLat/observerLon, updates slider, then disables it
    - Turbulencia: slider (0-10), only visible in advanced mode

Level 1 ‚Äî Sub-page (Ubicacion or Fecha y Tiempo):
  Header: [X back to menu] + page title text (other tab disappears completely)
  Body: GlobePicker or DateTimePicker fills the space


COMMON MODIFICATIONS
---------------------
To add a new lens:         Edit LENSES array in AdvancedMode.svelte
To add a new stel option:  Add to STEL_BUTTONS in stores.js + add button in Menu.svelte
To change location:        Modify default in initializeStelEngine() in stellarium.js
To add a new message:      Add handler in protobject.js (viewer or telescope side)
To change orientation:     Edit orientation.js (thresholds, gains, smoothing)
To change zoom range:      Edit MIN_FOV/MAX_FOV in stores.js + sliderToFov in fov.js
To add a new page:         Add to config.js + register handler for its messages in protobject.js
To change seeing shader:   Edit seeing-overlay.js (WebGL fragment shader)
To change throttle rates:  Edit constants in stores.js
To add keyboard shortcuts: Create new handler in Telescope.svelte (arduino removed, HID planned)
